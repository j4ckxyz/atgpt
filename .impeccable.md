## Design Context

### Users
Regular people, not developers. They sign in with a co/core API key (which resolves to an AT Protocol / Bluesky identity) and want to chat — same way they'd open ChatGPT. They have little interest in model IDs, routing, or network details. Everyday context: a quick question, writing help, thinking something through. A small fraction also run provider nodes but the interface should never assume that.

### Brand Personality
**Three words: warm, effortless, quietly characterful.**

Creative north star: "Daybreak" — the moment pale-blue sky meets first warm coral light. Calm, optimistic, human. A friendly place to talk where every new chat is a fresh, unintimidating start. Delight lives in small moments (a nice empty state, a satisfying send, character in copy), never in visual noise or clutter.

Anti-references (explicitly what this must NOT look like):
- Developer tools / terminals: no wall-to-wall monospace, no exposed model strings in main UI
- Crypto / web3 neon: no neon-on-black, glowing gradients, or hype styling
- Enterprise SaaS templates: no hero-metric blocks, identical icon-card grids, stock polished sameness
- Cluttered power-tools: settings and parameters must not crowd the conversation

### Aesthetic Direction
**Theme**: Light-leaning by default, with automatic dark mode via `prefers-color-scheme`. The "warm paper" background is an off-white tinted warm, never pure white. Dark variant is warm charcoal, never pure black.

**Color system (already implemented in OKLCH)**:
- Daybreak Blue `oklch(0.55 0.135 252)` — primary actions, send affordance, links, focus states
- Warm Coral `oklch(0.66 0.16 32)` — warmth, identity, delight moments; used sparingly (10% rule)
- Warm neutrals (paper, stone, ink) tinted toward the palette hues
- **The Two-Light Rule**: only these two hues carry meaning. No third accent (no green, no neon, no violet).
- **The Coral-Is-Rare Rule**: coral must cover only a small fraction of any screen.

**Typography**: Nunito (rounded humanist sans) for all UI. JetBrains Mono only inside rendered code blocks, never in UI chrome. Single-family system — hierarchy via size and weight only (at least 1.25× ratio between steps).

**Elevation**: Flat at rest. Soft shadows appear only as interaction feedback (focus on composer, hover on history items). No decorative glass/blur.

**Motion**: Responsive, not theatrical. Ease-out, no bounce. `prefers-reduced-motion` fully honored.

### Design Principles
1. **Effortless over capable-looking.** Newcomer should be chatting in seconds with no jargon or model-picker homework.
2. **The conversation is the product.** Settings, stats, and network details recede behind the message thread. Chrome earns its space or disappears.
3. **Personal by default, honest about privacy.** Greets by name, remembers chats, and is candid that memory lives in this browser only.
4. **Delight in the details, calm in the whole.** Character shows in moments, never in density.
5. **Invite participation, do not preach it.** The co-op depends on node operators — nudge gently, never block or moralize.

### Accessibility Target
WCAG 2.1 AA: sufficient text and UI contrast, full keyboard operability, labeled controls, visible focus. No color-only meaning. Comfortable tap targets for non-expert users.
